## 🚀 Как использовать

### Первый запуск

1. **Запустите backend:**
   ```bash
   cd backend
   pip install -r requirements.txt
   python -m uvicorn main:app --reload --host 0.0.0.0 --port 8000
   ```

2. **Запустите frontend:**
   ```bash
   cd frontend
   npm install
   npm run dev
   ```

3. **Откройте приложение:**
   - Перейдите на http://localhost:3000
   - Увидите экран регистрации/входа

### Регистрация

1. Нажмите "Зарегистрироваться"
2. Введите имя пользователя (минимум 3 символа)
3. Введите пароль (минимум 4 символа)
4. Нажмите "Зарегистрироваться"

### Вход

1. Введите имя пользователя
2. Введите пароль
3. Нажмите "Войти"

### Выход

Нажмите на кнопку с вашим именем в правом верхнем углу

## 🔒 Безопасность

- Пароли хешируются с помощью bcrypt
- Токены сессий генерируются криптографически безопасным способом
- Каждый пользователь видит только свои чаты
- API защищен от несанкционированного доступа

## 📁 Структура данных

### Backend
```
backend/
├── users/              # Данные пользователей
│   └── username.json   # Хешированный пароль и метаданные
├── chats/              # Чаты пользователей
│   └── username_chatid.json  # Чат привязан к пользователю
└── sessions.json       # Активные сессии
```

### Формат пользователя
```json
{
  "username": "john",
  "password": "$2b$12$...",  // bcrypt hash
  "created_at": "2026-02-19T..."
}
```

### Формат чата
```json
{
  "id": "1234567890",
  "username": "john",  // Владелец чата
  "title": "Название чата",
  "messages": [...],
  "model": "Flickers AI 2.5 FAST",
  "updated_at": "2026-02-19T..."
}
```

## 🔧 API Endpoints

### Авторизация
- `POST /api/auth/register` - Регистрация
- `POST /api/auth/login` - Вход
- `POST /api/auth/logout` - Выход
- `GET /api/auth/me` - Текущий пользователь

### Чаты (требуют авторизации)
- `GET /api/chats` - Список чатов пользователя
- `GET /api/chats/{chat_id}` - Получить чат
- `POST /api/chats/save` - Сохранить чат
- `DELETE /api/chats/{chat_id}` - Удалить чат

## 🌐 Деплой на Render/Vercel

### Render (Backend)
1. Добавьте переменные окружения:
   - `HUGGINGFACE_API_KEY` - ваш ключ

2. Render автоматически установит зависимости из `requirements.txt`

### Vercel (Frontend)
1. Добавьте переменную окружения:
   - `VITE_API_URL` - URL вашего backend на Render

2. Vercel автоматически соберет проект

## 🐛 Troubleshooting

### Ошибка "Не авторизован"
- Проверьте что токен сохранен в localStorage
- Попробуйте выйти и войти заново

### Чаты не загружаются
- Проверьте что backend запущен
- Проверьте консоль браузера на ошибки
- Убедитесь что вы авторизованы

### Ошибка при регистрации
- Имя пользователя должно быть минимум 3 символа
- Пароль должен быть минимум 4 символа
- Имя пользователя должно быть уникальным

## 📝 Примечания

- Токены сессий хранятся в `sessions.json` на backend
- При перезапуске backend сессии сохраняются
- Пользователи могут иметь несколько активных сессий
- Чаты автоматически привязываются к пользователю при сохранении
